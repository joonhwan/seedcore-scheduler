# 간트차트 이미지 내보내기 — 설계 문서

- 작성일: 2026-07-20
- 대상: SAM Scheduler 타임라인 뷰(`apps/web/src/pages/ProjectTimelinePage.tsx`, `apps/web/src/components/Timeline.tsx`)
- 상태: 설계 검토 대기, 구현 계획 대기

## 1. 목표

간트차트를 PPT·문서에 넣을 수 있도록 **PNG 이미지로 내보내는 기능**을 추가한다.

- 타임라인 뷰 헤더에 **"Export" 드롭다운 버튼**을 두고, 첫 항목으로 "이미지로 내보내기 (PNG)"를 제공한다.
- 캡처 대상은 **간트 차트 영역만**(왼쪽 트리 라벨 + 오른쪽 막대). 페이지 헤더·우측 상세 패널은 이미지에 넣지 않는다.
- 기본은 **모든 노드를 펼친 상태**로 출력하되, 대화상자에서 **"N단계까지만 펼침"**을 고를 수 있고 그 깊이대로 이미지가 나온다.
- **오늘 날짜선은 이미지에서 제거**한다.
- 화면에서 보던 색·막대 스타일을 그대로 유지한다.

## 2. 조사로 확인된 사실

- 타임라인은 SVG·canvas가 아니라 **순수 DOM + Tailwind**로 그려진다. `Timeline.tsx`.
- 접기/펼치기는 `collapsedIds`(Set)로 관리하며, `expandAll()`(233~235줄)은 이 Set을 비운다. `collapseAll()`(228~231줄)은 모든 GROUP을 넣는다.
- `flattenTree(items, collapsedIds)`(67~80줄)는 **접힌 그룹의 자식을 순회에서 제외**한다. 화면에 보이는 행 목록이 이 함수 결과다.
- 오늘 날짜선은 `todayInRange && <div className="… bg-rose-500" />`(955~961줄) **한 곳**뿐이다.
- 눈금 단위(일/주/월/분기)는 사용자가 직접 고르는 값이 아니라 **하루당 픽셀(`ppd`)에서 자동 파생**된다(328~333줄). 단위별 기본 배율은 `PPD` 상수(51~56줄): 일=36, 주=10, 월=4, 분기=2 (px/일).
- 날짜→픽셀: 가로 폭 = `전체일수 × ppd`, 막대 위치 = `dayDiff(start, range.start) × ppd`. 행 높이 `ROW_HEIGHT=32`, 헤더 높이 `HEADER_HEIGHT=44`(58~59줄).
- `range`는 실제 일정 최소~최대에 **앞뒤 1년씩** 여유를 붙인다(1329~1333줄). 화면맞춤(`fitToScreen`, 584줄)은 이 여유를 빼고 실제 일정 범위만 화면에 맞춘다.
- 테마는 `<html>` 요소에 `dark` 클래스를 붙였다 떼는 방식이다(`lib/theme.ts` 24~25줄). Tailwind `dark:` 스타일이 이 조상 클래스를 본다.
- 트리 깊이는 `depth` 0~4, 최대 5단계(AGENTS.md 4.6). 각 노드는 `depth` 필드를 가진다.
- `apps/web`에는 DOM을 이미지로 바꾸는 라이브러리가 **없다**(직접 확인). 신규 설치가 필요하다.

**결론: 백엔드·DB·공용 스키마 변경이 전혀 없다. 프론트엔드 표시·내보내기만 추가한다.**

- **예상 크기 안내**: 선택에 따라 "약 W × H px"를 실시간으로 다시 계산해 보여준다.
- **한계 경고**: 한 변이 안전 한계(16,000px)를 넘으면 빨간 경고를 띄우고 [내보내기] 버튼을 비활성화한다.
- 안내 문구: "선택한 깊이까지 펼쳐지고, 오늘 날짜선은 빠집니다."
- 하단: [내보내기] [취소].

## 5. 펼침 깊이 규칙 (추가 요구)

- 트리 깊이는 `depth` 0~4. 사용자에게 보이는 표기 **"K단계" = `depth` K-1**로 매핑한다.
  - 1단계 = 최상위 노드만(depth 0)
  - 2단계 = 최상위 + 그 직속 자식(depth 0~1)
  - …
- **"전체"** = `collapsedIds`를 비운다 → 모두 펼침(기본 동작).
- **"K단계까지"** = `depth >= K-1`인 **GROUP**을 `collapsedIds`에 넣는다 → depth 0~K-1만 보이고 그 아래는 접힌다.
  ```
  collapsedIds = items
    .filter(n => n.kind === 'GROUP' && n.depth >= K - 1)
    .map(n => n.id)
  ```
- 대화상자의 단계 선택지는 **프로젝트 트리의 실제 최대 깊이까지만** 제공한다(`max(depth) + 1` 단계). 그보다 깊은 단계를 고르는 것은 "전체"와 결과가 같아 무의미하기 때문이다.
- 이 `collapsedIds`를 정적 렌더의 `flattenTree`에 그대로 넘긴다. 따라서 **이미지에 실제로 보이는 행이 선택 깊이대로** 결정되고, 세로 크기·예상 크기 계산도 이 결과 행 수를 쓴다.

## 6. 이미지 생성 절차

1. **화면 밖(off-screen)에 정적 전용 간트를 잠깐 렌더**한다:
   - 5절 규칙으로 만든 `collapsedIds`(선택 깊이 반영)
   - 선택한 눈금의 기본 배율(`PPD` 상수)로 **전체 기간**(앞뒤 1년 여유 제외, 화면맞춤과 같은 실제 일정 범위)
   - 오늘 날짜선 없음, 스크롤 잘림 해제(전체 크기 그대로 펼침), hover·드래그·편집 UI 없음
   - 선택한 테마 클래스 적용, **배경색 명시**(라이트=흰색, 다크=`slate-900`)
2. 그 요소를 `html-to-image`의 `toPng`로 캡처한다(`pixelRatio: 2`).
3. 만들어진 PNG를 내려받고, 화면 밖 렌더를 제거한다.
4. **파일명**: `{프로젝트명}_간트_2026-07-20.png` 형태(내보내는 날짜).

## 7. 크기 계산 규칙

- **가로 폭** = 트리 라벨 폭 + (실제 일정 전체 일수 × 선택 눈금 기본 배율)
- **세로 높이** = 헤더(44px) + (선택 깊이 반영한 보이는 노드 수 × 32px)
- **최종 픽셀** = 위 값 × 2배(해상도)
- 대화상자에서 이 값을 미리 계산해 보여주고, 한 변이 16,000px를 넘으면 경고한다.
  - 참고 규모감(2배 기준): 1년 프로젝트를 월 눈금으로 = 가로 약 2,900px. 노드 100개 펼침 = 세로 약 6,500px.

## 8. 컴포넌트 구조 — 정적 전용 컴포넌트 (추천)

- 스크롤·드래그·내부 상태(`ppd`, `collapsedIds`, 자동 스크롤 등)가 얽힌 기존 `Timeline`(약 1,600줄)에 export 분기를 더하지 않는다.
- 대신 **정적 렌더 전용 컴포넌트**를 새로 두고, 순수 계산·렌더 로직만 공유한다:
  - 공유 후보: `computeRange`, `computeHeaderCells`, `buildTree`/`flattenTree`, 막대·라벨의 픽셀 계산.
- 어떤 함수·조각을 공용 파일로 뺄지는 구현 계획에서 확정한다(기존 `Timeline`의 겉보기 동작은 바꾸지 않는다).

## 9. 의존성 (폐쇄망 대응)

- `apps/web`에 `html-to-image`를 **dependency로 설치**한다(devDependency 아님).
- 빌드 산출물에 함께 담기므로 외부 접속 없이 폐쇄망에서 동작한다.
- DB·마이그레이션·공용 스키마 변경은 없다.
