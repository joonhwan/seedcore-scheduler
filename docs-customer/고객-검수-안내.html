SAM Scheduler · 고객 제안 / 검수 안내

팀의 일정을 한 그루의 나무로,
진행 상황은 한 장의 타임라인으로.

SAM Scheduler 는 인터넷이 닿지 않는 사내 서버에서 돌아가는 프로젝트 일정관리 도구입니다. 일정을 그룹으로 묶어 트리로 정리하면, 상위 일정의 기간과 진행율은 자동으로 계산되고, 전체 흐름은 타임라인 한 화면에서 읽힙니다.

사내망 완결 · 외부 연결 불필요 매일 자동 백업 · 새벽 4시 모든 변경 기록 · 누가 · 언제 · 무엇을 설치 한 번 · 단일 서버 패키지
At a glance

이 도구로 할 수 있는 일

복잡한 기능 대신, 일정관리에 꼭 필요한 네 가지에 집중했습니다.

일정을 트리로 정리

큰 단계(그룹) 아래 작은 일감(항목)을 최대 5단계까지. 순서 바꾸기, 다른 그룹으로 옮기기도 버튼으로 끝.

타임라인으로 한눈에

일 · 주 · 월 · 분기 단위로 전체 일정을 조망. 막대가 채워진 만큼이 진행율이라 회의 자료로 바로 씁니다.

진행율은 저절로

항목마다 % 만 입력하면 그룹과 프로젝트 전체의 진행율·기간은 자동 계산. 상위 일정을 따로 챙길 필요가 없습니다.

기록과 백업은 자동

누가 무엇을 바꿨는지 전부 이력으로 남고, 매일 새벽 데이터 전체가 자동 백업됩니다. 실수해도 되돌릴 길이 있습니다.

Architecture

인터넷 없이, 사내에서 완결

모든 것이 사내 서버 한 대 안에서 돌아갑니다. 외부 서비스 호출이 한 건도 없어서 보안망 안에서 안심하고 쓸 수 있고, 설치도 반입 패키지 하나로 끝납니다.

서버 1대 · 설치 패키지 1개 · 외부 호출 0건 — 도입 절차가 단순합니다.

Roles

세 가지 역할이면 충분합니다

복잡한 권한표 없이, 실제 팀 구조 그대로 세 역할만 기억하면 됩니다.

ADMIN

시스템 관리자

계정을 만들고, 비밀번호를 리셋하고, 프로젝트를 개설합니다. 필요할 때 "관리자 모드"로 어느 프로젝트든 정리할 수 있고, 그 편집은 전부 별도 기록으로 남습니다.

MANAGER

프로젝트 담당자

자기 프로젝트의 멤버를 넣고 빼고, 프로젝트를 보관(종료) 처리합니다. 물론 일정 편집도 자유롭게.

MEMBER

팀원

참여 중인 프로젝트의 일정을 만들고 고치고, 진행율을 올리고, 댓글로 소통합니다. 자기와 무관한 프로젝트는 아예 보이지 않습니다.

Screen tour

화면으로 보는 사용 흐름

아래 화면은 모두 실제 동작 중인 v1 을 그대로 캡처한 것입니다 (다크 모드).

01

로그인 — 아이디와 비밀번호면 끝

메일 인증도, 외부 가입 절차도 없습니다. 관리자가 계정을 만들어 주면 바로 시작합니다.

할 수 있는 것

  • 사내 계정으로 로그인 — 임시 비밀번호로 처음 들어오면 비밀번호 변경부터 안내
  • 화면 우측 상단에서 라이트 / 다크 테마 전환

안심되는 점

  • 비밀번호 5회 연속 오류 시 15분 자동 잠금 — 무차별 시도를 차단
  • 자리를 비우면 세션이 자동 만료 (30분 유휴, 최대 12시간)
02

프로젝트 목록 — 내 것만 보입니다

로그인하면 내가 참여 중인 프로젝트 카드만 나타납니다. 상태·멤버 수·내 역할이 카드에서 바로 읽힙니다.

할 수 있는 것

  • 카드 클릭 한 번으로 프로젝트 안으로
  • 관리자는 "관리자 모드"를 켜서 모든 프로젝트를 열람·정리

간편한 점

  • 남의 프로젝트가 목록에 섞이지 않아 찾을 필요가 없음
  • 관리자 모드가 켜져 있으면 상단에 띠 배너로 명확히 표시 — 실수로 켜둔 채 잊는 일 방지
03

일정 트리 — 계획을 접었다 펴듯 관리

왼쪽은 일정 구조, 오른쪽은 선택한 일정의 상세. 큰 단계는 그룹(G)으로 묶고, 실제 일감은 항목(I)으로 만듭니다.

할 수 있는 것

  • 일정 추가·수정·삭제, 위아래 순서 바꾸기, 다른 그룹으로 옮기기 — 전부 버튼으로
  • 진행율은 슬라이더로 5% 단위 입력
  • 일정마다 댓글로 논의, 변경 이력은 자동 기록

간편한 점

  • 이력에 "진행율 0 → 60" 처럼 바뀐 내용이 그대로 남아 보고가 필요 없음
  • 두 사람이 같은 일정을 동시에 고치면 나중 저장자에게 즉시 알림 — 덮어쓰기 사고 방지
  • 잘못된 이동(순환 구조, 5단계 초과)은 시도 전에 미리 막아줌
04

그룹은 손대지 않아도 저절로 맞춰집니다

그룹의 시작일·종료일·진행율은 입력하는 값이 아니라 하위 항목에서 자동으로 계산되는 값입니다. 항목만 최신으로 유지하면 상위 보고 숫자는 항상 맞습니다.

자동으로 되는 것

  • 기간 = 하위 항목들의 가장 빠른 시작일 ~ 가장 늦은 종료일
  • 진행율 = 하위 항목들의 평균 (화면에 "자손 ITEM 평균"으로 표기)

간편한 점

  • 일정이 늘어나거나 줄어도 상위 계획을 고칠 일이 없음
  • 그룹에 직접 입력하려는 시도는 화면에서부터 막혀 있어 숫자가 어긋날 여지가 없음
05

타임라인 — 보고용 화면이 따로 필요 없습니다

트리에서 관리한 일정이 그대로 간트 차트가 됩니다. 막대가 채워진 만큼이 진행율이라, 이 화면 하나로 주간회의가 끝납니다.

할 수 있는 것

  • 일 / 주 / 월 / 분기 단위 전환 — 이번 주 실무부터 분기 로드맵까지
  • "오늘로 이동" 버튼, 오늘 위치는 붉은 선으로 표시
  • 막대를 클릭하면 우측 패널에서 바로 수정 (트리 화면과 동일)

간편한 점

  • 별도 엑셀 간트를 만들고 갱신하는 일이 사라짐 — 데이터는 한 곳, 뷰는 두 개
  • 그룹(보라)과 항목(파랑)이 색으로 구분되어 구조가 한눈에
06

사용자 관리 — 계정 업무가 1분이면 끝

관리자 화면에서 계정 생성부터 비밀번호 리셋, 잠금 해제까지 전부 처리합니다. IT 부서에 요청서를 쓸 일이 없습니다.

할 수 있는 것

  • 계정 생성 → 임시 비밀번호 전달 → 본인이 첫 로그인에서 변경
  • 비밀번호 리셋, 5회 오류 잠금 해제, 퇴사자 비활성화
  • 검색·상태 필터, 표시 이름은 그 자리에서 바로 수정

안심되는 점

  • 임시 비밀번호는 발급 직후 한 번만 표시 — 어디에도 저장되지 않음
  • 비활성화하는 순간 그 사용자의 접속이 즉시 끊김
  • 계정 관련 조치는 전부 기록으로 남음
07

멤버 관리 — 담당자가 직접, 검색해서 추가

프로젝트 담당자(MANAGER)가 팀원을 검색해 역할과 함께 추가합니다. 관리자를 거칠 필요가 없습니다.

할 수 있는 것

  • 사용자 검색 → 역할(담당자/팀원) 선택 → 추가, 세 번의 클릭
  • 프로젝트를 떠난 인원은 한 번에 제외

안심되는 점

  • 마지막 남은 담당자는 제거할 수 없게 막혀 있어 주인 없는 프로젝트가 생기지 않음
Behind the scenes

사람이 잊어도, 시스템은 잊지 않습니다

운영자가 매일 해야 할 일을 없애는 것이 목표였습니다. 백업과 기록은 전부 자동입니다.

모든 변경에 발자국

로그인 · 일정 수정 · 멤버 변경 · 계정 조치 — 누가, 언제, 어떤 값에서 어떤 값으로 바꿨는지 자동 기록됩니다. 관리자 모드로 한 편집은 별도 표시까지.

동시 수정 걱정 없이

두 사람이 같은 일정을 열어 고치면, 나중에 저장한 쪽에 "다른 사람이 먼저 수정했습니다" 알림이 떠서 모르고 덮어쓰는 사고를 막습니다.

실수 방지 장치

활성 프로젝트는 삭제 자체가 안 되고(보관 후에만 가능), 마지막 담당자 제거·5단계 초과·순환 구조 같은 위험한 조작은 전부 사전에 거부됩니다.

Scope

v1 에 담지 않은 것 — 솔직하게

의도적으로 뺀 기능입니다. 대부분 확장 자리를 미리 마련해 두었고, 검수 의견에 따라 다음 버전 우선순위를 정합니다.

타임라인 막대 드래그로 날짜 조정다음 버전(v1.x) 예정 — 지금은 막대 클릭 후 우측 패널에서 수정합니다.
트리 드래그&드롭버튼 이동(위/아래)과 "부모 변경" 창으로 같은 일을 할 수 있습니다.
캘린더 뷰 · 첨부파일 · 선후행 연결 · 반복 일정v1.1 이후 검토 — 데이터 구조에는 자리를 만들어 두었습니다.
메일 · 메신저 알림폐쇄망 특성상 v1 미포함 — 사내 알림 시스템이 도입되면 연동할 수 있게 준비되어 있습니다.
사내 계정(LDAP/SSO) 연동v1 은 자체 계정 — 연동용 확장 지점은 마련되어 있습니다.
다국어v1 은 한국어 — 영어 등 추가가 쉬운 구조로 만들어져 있습니다.
